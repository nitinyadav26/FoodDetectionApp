# FoodSense Screenshot Requirements

## Required Screenshots

Capture the following screens for both iOS and Android store listings:

### 1. Dashboard
- Daily calorie ring and macro breakdown
- XP progress bar and current level/title
- Streak counter visible
- Light mode preferred for primary screenshot

### 2. Food Scan (AI Detection)
- Camera view with food detected
- Nutrition overlay showing calories, protein, carbs, fat
- Food quality score badge visible
- Use a visually appealing Indian or international dish

### 3. AI Nutritionist Chat
- Conversation with AI coach showing personalized advice
- At least 3-4 messages visible in the thread
- Show a mix of user questions and AI responses

### 4. Social Tab
- Activity feed with friend posts and emoji reactions
- Friends list or challenge in progress
- Leaderboard ranking visible if possible

### 5. Badges Collection
- Badge grid showing unlocked and locked badges
- Multiple categories visible (Streak, Logging, Nutrition, Social)
- XP level and title displayed at top

### 6. Meal Plan
- AI-generated 7-day meal plan view
- Individual day expanded showing breakfast, lunch, dinner
- Calorie totals per meal visible

## iOS Screenshot Sizes

| Device | Size (pixels) | Required |
|--------|--------------|----------|
| iPhone 15 Pro Max / 16 Pro Max (6.7") | 1290 x 2796 | Yes |
| iPhone 15 Pro / 16 Pro (6.1") | 1179 x 2556 | Yes |
| iPad Pro 12.9" (6th gen) | 2048 x 2732 | Yes |

**Format:** PNG or JPEG, sRGB color space, no alpha channel
**Count:** Minimum 3, maximum 10 per device size
**Orientation:** Portrait preferred, landscape acceptable for iPad

## Android Screenshot Sizes

| Device | Size (pixels) | Required |
|--------|--------------|----------|
| Phone | 1080 x 1920 minimum (16:9) or 1080 x 2400 (20:9) | Yes |
| 7" Tablet | 1200 x 1920 | Yes |
| 10" Tablet | 1600 x 2560 | Yes |

**Format:** PNG or JPEG, 24-bit, no alpha
**Count:** Minimum 2, maximum 8 per device type
**Orientation:** Portrait preferred

## Tips

- Use demo/sample data that looks realistic (not empty states)
- Ensure the status bar shows full signal, Wi-Fi, and a reasonable time
- Hide any debug indicators or developer settings
- Test in both light and dark mode; use light mode for primary screenshots
- For iPad screenshots, ensure the layout adapts properly (no stretched phone UI)
- Feature graphic (Android): 1024 x 500 PNG, used as banner in Play Store listing
